Definición de Formularios en HTML

Los formularios en HTML, son elemenos que nos va a dotar de cierta funcionalidad y dinamismo a nuestras páginas web.
Nos van a permitir interactuar con nuestras webs, por lo tanto, son elementos fundamentales para poder recoger
información.
Los formuarlios por lo tanto, nos permitirán como principal función, el recopilar información para poder
gestionarla y almacenarla según nuestro fin.
Funciones principales:
  • Recoger información del usuario
  • Enviar esos datos que se han guardad con el fin de poderlos procesasr o realizar cualquier
    tipo de operación que se requiera con ellos.

Los principales objetivos que tenemos que tener en cuenta a la hora de diseñar un formulario en HTML,
son los siguientes:
  • El formulario tiene que ser lo más sencillo posible a la hora de insertar datos por parte del usuario
  • La experiencia usario /usabilidad (UX), tiene que ser lo más agradable posible
  • Los datos introducidos tienen que tener un formato predecible y esperado.
  • En la medida de lo posible, reducir errores posibles que se puedan dar al introducir los datos.
    Para ello, comunicar al usuario si esos datos son correctos o no, o si se están insertando correctamente.

Etiquetas básicas para la creación de un Formulario en HTML

Para crear un formularo en HTML, utilizaremos como base la etiqueta form de apertura y de cierre.
Los elementos que tendremos que tener en cuenta además de los anteriormente comentados, son los siguientes:
  • action:Indicar la URL/URI o BBDD o correo etc. (Dirección Backend) a la que queremos Enviar
    la información de nuestro formuarlio.
  • Method:Método de envío de información. GET o POST. El método get, se utiliza para
    mostrar la información que se quiere enviar a través del protocolo HTTP/HTTPS de nuestra web.
    El método post, que funciona de la misma manera que get, sólo que oculta la información en
    la cabecera del navegador.
  • name: se utiliza para dar nomrbe al formulario
  • target: se utiliza para abrir el formulario en otra pestaña.P.E. target=_blank
  • entype: codificar información del envío del formulario. Es lo que se conoce como un token
  • autocomplete: Autocompleta la información de nuestro formulario a través del guardado de información
    en la caché del navegador o cookies

Tipos de datos que podemos gestionar en un Formulario

El tipo de información o datos que podemos gestionar en un formulario, es de distintos tipos.
Entre ellos podemos destacar los siguientes como los más utilizados y/o importantes:
  • Informarción de texto: Nombres, apellidos, direcciones físicas, etc. Utilizaremos la etiqueta input
    con el type = text. También podemos usar la etiqueta texarea. Esta última es una caja donde podemos
    insertar texto.
  • Números o cantaidades: Edades, precios, cuantías, etc. Utilizaremos el elemento input pero del tipo
    number (input type = number)
  • Fechas u horas: fechas de nacimiento, calendarios, etc. Utilizaremos el elemento input pero del tipo
    date, week, etc.
  • Opciones de verdadero o falso: Utilizaremos input type = checkbox
  • Opciones únicas: podremos elegir entre 2 o más opciones. input type = radio o select
  • Opciones única simples: donde utilizaremos select múltiple, input checkbox, etc.
  • Opciones de listas abiertas:datalist
  • Seleccionar colores: input type = color
  • Sekeccionar archivos: input type = file

Campos de Entrada de Información Numéricos

Para poder obtener información númerica en nuestros formularios, podemos utilizar dos tipos de campos:
  • input type =number:con el cuál estamos indicando que el valor de entrada será númerico
  • campo de entrada de rango: input type = range

Los atributos que van asociados a estos son los siguientes:
  • min: establece un valor mínimo para la validación
  • max: establece un valor máximo para la validación
  • step: permite la variación entre números para la posterior validación

Listas de Selección

Se utilizan cuando por ejemplo, las casillas de verificación se nos quedan cortas o necesitamos mostrar una lista
más extensa dedatos.
A través de estas lista, podemos mostrar al usuario varias opciones disponibles para elegir una de ellas.
Entre los tipos de listas podemos destacar las siguientes:
  • Listas Cerradas: Son listas de opciones cerradas. Se hacen con select y option
  • Listas Abiertas: Son listas de opciones abiertas. Se hacen con datalist

Agrupación de Campos

Son etiquetas que nos permiten organizar los elementos de nuestros formularios reagrupándolos por categorías o temáticas
entre otras cosas.
  • Fielset: agrupa de manera organizada los campos de entrada por temática
  • Legend: una leyenda para la etiqueta fieldset
  • Label: relaciona campo y texto de un formulario
